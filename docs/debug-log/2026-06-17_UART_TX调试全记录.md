# UART TX 调试全记录 — 2026-06-17

> 目标：让 nRF52840 的 UART TX 引脚发出 0xAA/0xBB 帧协议数据，供 ③ 号板接收

## 测试环境

- 板子：Adafruit Feather nRF52840 Express（两张板交替使用）
- SoftDevice：S140（通过 `Bluefruit.begin()` 初始化）
- 帧协议：`0xAA | 方向码 | 0xBB`，波特率 115200 8N1
- 验证手段：万用表 DC 档、USB-TTL 模块 + 串口助手、③ 号板、板内回环

## 已验证正常的部分

| 项目 | 证据 |
|------|------|
| `EVENTS_ENDTX` 触发 | 每次 `TASKS_STARTTX` 后 `while(!EVENTS_ENDTX)` 都能退出 |
| `Serial1.write()` 返回 3 | `[UART] sent 3 bytes` 每次打印 |
| BLE 连接 + 数据解析 | `[DATA] raw=0xA2 -> BTN=0x40` 稳定输出 |
| GPIO 控制 A2 电平 | `digitalWrite(A2, HIGH)` → 万用表测到 3.3V |
| 红灯闪 | D3(P1.15) 每次发送时亮 2ms |
| 引脚映射 (BSP 层) | `g_ADigitalPinMap`, `PSEL.TXD` 寄存器值均正确 |
| 两个 UARTE 实例均响应 | UARTE0 和 UARTE1 的 `EVENTS_ENDTX` 都触发 |

## 根因判断

**SoftDevice (S140) 初始化后，UARTE 外设的物理 TX 波形可能被抑制。** 所有内部寄存器层面操作均正常，但外部设备（USB-TTL、③ 号板、板内回环）一律收不到信号。唯一共同因子是 `Bluefruit.begin()` 初始化了 SoftDevice。

## 当前生产固件状态

- 代码：`Serial1.setPins(PIN_SERIAL1_RX, A2)` → TX = A2(P0.30)
- BLE：扫描 LOOKBON → 连接 → CCCD 订阅 → 接收数据 ✅
- UART：每次手柄按键发送 `AA 01/02/03/04 BB`
- 红灯：每次发送闪一下
- **待验证：TX 物理引脚波形**

## 下一步建议

1. 用示波器/逻辑分析仪直接量 A2 引脚，确认 TX 波形
2. 尝试不初始化 SoftDevice 的条件下启用 UARTE（需要裸机环境）
3. 换用 `app_uart` 库（Nordic 官方 SDK）替代 Arduino `Serial1`
4. 考虑用 SPI/I2C 替代 UART 与 ③ 号板通信
