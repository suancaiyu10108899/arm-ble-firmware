# 开发环境搭建指南

> **项目**：arm-ble — nRF52840 VR 手柄蓝牙接收固件
> **板子**：Adafruit Feather nRF52840 Express
> **最后更新**：2026-06-14

---

## 1. 安装 Python + PlatformIO

### Windows

```powershell
# 1. 安装 Python 3.x (如果还没装)
#    从 https://python.org 下载安装，勾选 "Add Python to PATH"

# 2. 安装 PlatformIO
pip install platformio

# 3. 验证
python -m platformio --version
```

### macOS / Linux

```bash
pip3 install platformio
python3 -m platformio --version
```

---

## 2. 克隆代码

```powershell
git clone https://github.com/suancaiyu10108899/arm-ble-firmware.git
cd arm-ble-firmware
```

---

## 3. 编译

```powershell
python -m platformio run --project-dir=.
```

首次运行会自动下载 nRF52 工具链（ARM GCC + nRF5 SDK），约 500MB，等待 5-10 分钟。

编译成功输出：

```
RAM:   [=         ]   6.2% (used 15536 bytes from 248832 bytes)
Flash: [==        ]  15.6% (used 127220 bytes from 815104 bytes)
======================================= [SUCCESS] ===
```

---

## 4. 烧录

```powershell
# 板子用 MicroUSB 数据线连接电脑
python -m platformio run --project-dir=. --target upload
```

### 烧录失败处理

| 错误 | 解决 |
|------|------|
| `PermissionError` | COM 口被串口监视器占用，先关掉 |
| `Timed out waiting for acknowledgement` | 按一下板子 Reset 按钮进入 Bootloader 模式重试 |
| `could not open port 'COMx'` | 换一根 MicroUSB 数据线（4芯，能传数据的那种） |

## 5. 串口监视

```powershell
python -m platformio device monitor --project-dir=. --baud 115200
```

预期输出：

```
=== ARM-BLE v2.7 (UART TX on A2) ===
[UART] Serial1 @ 115200 baud (TX=A2=P0.30, RX=D1=P0.24)
[OK] v2.7 ready
[FOUND] LOOKBON 0D:CE:99:03:5D:D2   ← 手柄在附近时自动出现
[CONNECT] conn=0
[OK] subscribed
[DATA] raw=0xA3 -> BTN=0x80 X=128 Y=128   ← 按键数据
[UART] sent 4 bytes: 80 80 80 00           ← UART 发出
```

按 `Ctrl+C` 退出。

## 常见问题

**Q: `python` 命令找不到？**
A: 用 `python3` 或 `py -3` 代替。确认 Python 已加入 PATH。

**Q: 需要 Arduino IDE 吗？**
A: 不需要。PlatformIO 自带编译器和库管理，一条命令搞定。

**Q: 怎么知道板子连在哪个 COM 口？**
A: `python -m platformio device list` 会列出所有可用的串口。
