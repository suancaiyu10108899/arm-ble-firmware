## 4. LOOKBON 手柄协议

- **数据是单字节编码**，不是 ASCII 字符串
- 高 4 位（nibble）= 事件类型，低 4 位 = 按键/方向编号

| 字节 | 含义 |
|------|------|
| `0xA1`-`0xA7` | 单击 @/A/B/C/D/R/L |
| `0xB1`-`0xB7` | 长按 |
| `0xC1`-`0xC7` | 释放 |
| `0xD0` | 摇杆中位 |
| `0xD1`-`0xD8` | 摇杆 8 方向 |

解析后输出三个变量（见 `src/handle_parser.cpp`）：
- `in.buttons` — 按键位掩码 (uint32)
- `in.joystickX` — 摇杆 X (0-255, 128=中)
- `in.joystickY` — 摇杆 Y (0-255, 128=中)

## 6. 常见问题

**Q: 烧录失败 "PermissionError"？**
A: COM 口被串口监视器占用，关掉监视器重试。

**Q: 手柄灯灭了但没数据？**
A: 手柄可能被其他蓝牙设备连走。关掉手机/PC 蓝牙重试。

**Q: `[DATA]` 全是 `BTN=0x0 X=128 Y=128`？**
A: 检查是否在跑 v2.6（不是旧版）。旧版 ASCII 解析器会导致此现象。
