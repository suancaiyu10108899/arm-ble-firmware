## 🎯 根因递进

### 扫描器 bug（v1.0-v1.4）
`Bluefruit.begin(peripheralCount, centralCount)` — **第一个参数是 Peripheral**。v1.3 全量日志证实 onScan 只触发一次后永久休眠，与 begin 参数无关。

### v1.5 突破：被动扫描 + loop 重启
`useActiveScan(false)` + loop 中每 10 秒 `Scanner.stop()` → `Scanner.start(0)` → 成功多次扫到 LOOKBON。

### GATT 发现不兼容（v1.6-v1.8）
Adafruit 库 `BLEClientService::discover()` → `BLEDiscovery::_discoverService()` → `sd_ble_gattc_primary_services_discover(..., &svc.uuid._uuid)` 传了 UUID 过滤，LOOKBON 不响应。改为 NULL 全量发现也无效——底层 GATT 协议交互不兼容。

### Python bleak 启示（v1.9-v2.0）
Python 参考代码 `ble_python.py` 跳过 discover，直接 `client.start_notify(7)`。v1.9 仿照用 `sd_ble_gattc_write` 往 handle 7/8 写 CCCD 成功，但 fake BLEClientCharacteristic 未关联 SoftDevice 事件分发。

### v2.0 方案
`Bluefruit.setEventCallback()` 注册原始 SoftDevice 事件回调，直接捕获 `BLE_GATTC_EVT_HVX` 通知。同时 `setInterval(32,16)` 快速扫描，精简 onScan 日志。
