## 固件代码

`src/main.cpp` — BLE UART 转发器:
- 广播名称: "ARM-BLE"（实际扫描显示为 N/A，属手机扫描端问题）
- 服务: Nordic UART Service (NUS)
- NUS UUID: `6E400001-B5A3-F393-E0A9-E50E24DCCA9E`
- 功能: 手机发任意字符 → 板子通过硬串口打印到电脑

## 测试步骤

1. 烧录 BLE UART 固件 (PlatformIO, COM6 → Bootloader COM8 → 成功)
2. 板子蓝灯**呼吸** = BLE 正在广播
3. 手机 nRF Connect → Scan → 找到 MAC `D3:52:88:3A:06:27` 带 NUS 服务的设备
4. **Connect** → 蓝灯变**常亮** = 连接成功
5. 点 TX 特征 → Send "Hello你好"
6. Arduino IDE 串口监视器打印 "Hello你好" ✅

## 踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| 第一次烧录 SUCCESS 但无蓝牙 | Bootloader 未正确接收数据（假成功） | 拔 USB 重插，重新烧录 |
| COM 口 6↔7↔8 来回变 | nRF52840 正常模式 vs Bootloader 模式各有独立 COM 号 | 每次烧录前用 `python -c "import serial.tools.list_ports..."` 确认当前 COM 号 |
| 设备名显示 N/A 而非 ARM-BLE | 手机扫描端 BLE 名称解析差异 | 通过 NUS UID `6E400001-...` 确认设备身份 |
| 串口监视器无法打开 | COM 口被其他程序占用 | 关闭所有串口工具后重新打开 |
